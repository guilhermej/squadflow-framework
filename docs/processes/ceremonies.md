# Ceremonies

The catalog of structured meetings in SquadFlow. Each has a purpose, a convener, and a defined output.

## Recurring

| Ceremony | Frequency | Duration | Convened by | Purpose / Output |
|---|---|---|---|---|
| **Debriefing** | daily (EOD) | ≤ 15 min (usually async) | Squad Lead | Each Player reports in a sentence what they did and what is blocked. Output: logged status, blockers surfaced for next Sync. |
| **Squad Sync** | weekly | ~30 min | Squad Lead | Alignment within a Squad — review Tasks, unblock, plan the week. Output: updated assignments, logged blockers, explicit priorities. |
| **Factory Review** | weekly (biweekly for slow Factories) | ~30 min | Factory Manager | Walk the kanban left-to-right. Output: stuck cards resurfaced, stage definitions refined, improvement proposals surfaced. |
| **Portfolio Review** | monthly | ~60 min | Org Steward | Stewards + Leads review all active Factories and Projects. Output: portfolio decisions (pause, cancel, form). |
| **OKR Setting + Review** | quarterly | ~90 min per session | OKR Sponsors + Steward | Two sessions bracketing the quarter — set new OKRs, score ending ones. Output: committed/closed OKRs. |
| **Strategic Cadence** | quarterly | ~120 min | Org Steward | Review direction; feed OKR Setting with strategic narrative. Output: strategic context for next period. |
| **Annual Planning** | yearly | full day | Org Steward | Year-long OKRs and strategic horizon. Output: annual OKRs and priorities. |

## Ad-hoc (triggered by lifecycle transitions)

| Ceremony | Trigger | Duration | Output |
|---|---|---|---|
| **Project Kickoff** | Project `backlog → active` | 30–60 min | Project is `active`; first Tasks on the board; kickoff notes. |
| **Project Closing / Retrospective** | Project → `closed` | 60 min | `close_reason` recorded; decision-log entry; retrospective notes. |
| **Squad Formation** | Squad `forming → active` | 60 min | Squad is `active`; mandate documented; first Squad Sync scheduled. |
| **Squad Dissolution Handoff** | Squad `dissolving → dissolved` | 60 min | Every Factory, Project, and open Task re-homed or closed. Decision-log entry. |
| **Factory Retirement Review** | Factory `active → retired` | 45 min | `retired` state set; decision-log entry with rationale; historical metrics preserved. |

## Principles

- **Every ceremony has a Role that convenes.** If the Role is vacant, the ceremony does not run.
- **Every ceremony has an output.** A meeting that produces nothing anyone references afterward is waste.
- **Shortest useful interval.** Daily for Player coordination, weekly for Squad, quarterly for strategy. Do not add cadences lightly.

## Minimum viable calendar

For < 30 people, run only:

- **Daily:** Debriefing (when Squads have ≥ 3 members).
- **Weekly:** Squad Sync per Squad; Factory Review per Factory.
- **Quarterly:** OKR Setting + Review.
- **Ad-hoc:** Kickoff and Closing as Projects demand.

Add Portfolio Review, Strategic Cadence, and Annual Planning as the organization grows.

## Antipatterns

- **Status theatre.** Meetings to read what is already visible. Cut.
- **Agenda inflation.** Inviting observers until the meeting is 15 people erodes candor.
- **Skipping without a reason.** "Things feel fine" is not a reason to skip a Factory Review. If it truly does not add value, retire it formally.
- **Inventing new cadences.** The default answer is no. First check if an existing one can absorb the need.
